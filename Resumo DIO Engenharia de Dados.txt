Relacional - é um banco onde vamos guardar informações que tenham relações entre si, com consistencia e esquema de dados rigido exmplo sistema transacional
e-comerce e antes de modelar vamos saber o esquema bem rigido, traz garantia das relações de dados de forma integra.

Não Relacional - surgiu o conceito NoSql não para substituir mas como alternativa, contar com capacidades adicionais, surgiu por necessidade de escalabilidade 
e de forma não extruturada, este mantem extrutura nao rigida que ate não conheça, com uma gama de possibilidades bem maior e complementa, permite trabalhar 
sem esquemas de dados rigido.

A ideia não é substituir os relacionais, mas sim complementar, e resolver problemas que os bancos relacionais não conseguem, com escalabilidade horizontal, 
e não rigidez de quelquer forma e jeito sem quebra de compatililidade e a consulta fica bem mais simples e armazenar de acordo com a consulta, e de forma simples
assim veio apenas para ajudar.

Como encontrar algo que não tem garantia de extrutura? para prever isso sempre devemos ter cuidados basicos com a chaves de consulta, pos podemos ter campos nulos, 
mas tem que ser  pensado basicamente para não pensar de uma unica forma, e pensar nas caracteristicas do banco para que possamos entregar performance, a aplicação
saber lhe dar com essa imprevisibilidade, mesmo assim será necessário modelar minimamente não se pode inserir os dados de forma indiscriminada.
